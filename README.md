## Repository layout

```text
.
├── decks/                 # Slidev source decks and published presentation artifacts
├── catalog/               # Generated deck + slide metadata indexes
├── site/                  # Minimal GitHub Pages shell / redirect
├── tools/                 # Validation, catalog, extraction, compose, and build helpers
├── tests/                 # Repo verification and smoke tests
└── .github/workflows/     # CI and GitHub Pages publication workflows
```

## Quick start

- `npm run validate` — validate manifests, slide frontmatter, taxonomy, and dependencies
- `npm run build:catalog` — regenerate deck and slide indexes
- `npm run build:site` — build the lightweight GitHub Pages output under `site/dist/`
- `npm run build` — rebuild catalog and site
- `npm test` — run the repo test suite
