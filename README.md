## Who This Is For

These decks are designed for solutions architects, platform engineers, and sales engineers who need to explain, implement, or operate Tyk API management capabilities. Foundation decks support onboarding and workshops; deep-dive decks support technical enablement, customer architecture discussions, and hands-on platform work.

## Catalog Metadata

Every deck is described with consistent metadata: topic, audience, level, use case, status, product tags, prerequisites, related decks, and publication settings. Every reusable slide is also indexed separately with its own title, summary, tags, standalone safety, dependency links, recommended next slides, and source file path.

| Metadata | Purpose |
| --- | --- |
| Topic | Main content area such as API Gateway, Security, GraphQL, gRPC, Streaming, Kubernetes, or Observability. |
| Audience | Primary learner persona: Solutions Architect, Platform Engineer, or Sales Engineer. |
| Level | Training depth: Fundamentals, Concepts, or Deep Dive. |
| Use case | How the material is intended to be used, commonly onboarding, workshop, or discovery. |
| Status | Publication lifecycle for the deck: active, draft, deprecated, or archived. |
| Slide metadata | The slide-level index supports searching, reuse, dependency tracking, and customer-safe subdeck composition. |

## Repository Layout

```text
.
|-- decks/                 # Slidev source decks plus generated presentation artifacts
|-- catalog/               # Generated deck, slide, taxonomy, and dependency indexes
|-- site/                  # Minimal GitHub Pages shell that redirects to this README
|-- tools/                 # Validation, catalog, extraction, composition, and build helpers
|-- tests/                 # Metadata, catalog, composition, site, and build verification
`-- .github/workflows/     # CI and GitHub Pages publication workflows
```

## Local Development

- `npm run validate` validates manifests, slide frontmatter, taxonomy values, and slide dependencies.
- `npm run build:catalog` regenerates `catalog/decks-index.json`, `catalog/slides-index.json`, and `catalog/dependency-graph.json`.
- `npm run build:decks` builds published Slidev decks into each deck `.artifacts/presentation` folder.
- `npm run build:site` builds the lightweight GitHub Pages output under `site/dist/`.
- `npm run build` rebuilds catalog, deck presentations, and site output.
- `npm test` runs the repository test suite.

## Publishing Notes

GitHub Pages publishes `site/dist/` from the `main` branch workflow. Published decks are selected from deck manifests where `publishing.github_pages` is `true` and the deck status is not `archived`.
